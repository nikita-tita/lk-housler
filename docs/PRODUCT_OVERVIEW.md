# LK Housler — Продуктовый обзор

## Что это

**LK Housler** — платформа для работы с клиентами в сфере недвижимости, которая автоматизирует расчёты между агентами, агентствами и клиентами с прозрачными платежами через T-Bank.

---

## Проблема рынка

| Боль | Как сейчас | Риск |
|------|-----------|------|
| Оплата "в конверте" | Наличные после сделки | Нет гарантий, споры, суды |
| Раздел комиссии | Устные договорённости | Конфликты между агентами |
| Договоры | Word-файлы, подпись лично | Долго, дорого, нет юр. силы |
| Учёт сделок | Excel, бумага | Потеря данных, нет аналитики |

---

## Решение

### Bank-Led Safe Deal — защищённые платежи через T-Bank

```
Client → T-Bank (hold) → [Service confirmed] → Agent/Agency
                       → [Dispute] → Refund/Arbitration
```

**Как работает:**
1. Агент создаёт сделку, указывает условия
2. Клиент получает ссылку, оплачивает через СБП/карту
3. Клиент оплачивает услугу напрямую исполнителю через T-Bank
4. Hold period защищает клиента до подтверждения оказания услуги
5. После подтверждения — банк распределяет средства участникам

**Гарантии:**
- Hold period для защиты клиента
- Подтверждение оказания услуги перед выплатой
- Dispute resolution через агентство/платформу
- Прозрачное распределение по договору

## Ключевые сценарии

### Сценарий 1: Простая сделка (1 агент)

```
Квартира: 10 млн руб.
Комиссия: 3% = 300 000 руб.
Платформа: 4% от комиссии = 12 000 руб.
────────────────────────────
Агент получит: 288 000 руб.
```

### Сценарий 2: Совместная сделка (2 агента)

```
Комиссия: 300 000 руб.
Агент 1 (60%): 180 000 руб.
Агент 2 (40%): 120 000 руб.
Платформа (4%): 12 000 руб.
────────────────────────────
Выплаты: 168 000 + 120 000 = 288 000 руб.
```

### Сценарий 3: Агент в агентстве

```
Комиссия: 300 000 руб.
Агентство (30%): 90 000 руб.
Агент (70%): 210 000 руб.
Платформа (4%): 12 000 руб.
────────────────────────────
Выплаты автоматически на счета обоих
```

## Юридическая защита

### Электронная подпись (ПЭП)

Соответствует 63-ФЗ "Об электронной подписи":
- Клиент получает SMS с кодом
- Вводит код на странице подписания
- Фиксируется: IP, время, хэш документа, User-Agent

**Юридическая сила:** Простая электронная подпись признаётся судами при наличии доказательной базы.

### Типы договоров

| Код | Договор | Стороны |
|-----|---------|---------|
| TPL-001 | Услуги по подбору (покупка) | Агент — Покупатель |
| TPL-002 | Услуги по продаже | Агент — Продавец |
| TPL-003 | Услуги по аренде | Агент — Арендатор/Арендодатель |
| TPL-004 | Эксклюзивный договор | Агент — Продавец |
| TPL-005 | Раздел комиссии | Агент — Агент |
| TPL-006 | Агентский договор | Агентство — Агент |

### Согласия (Compliance)

| Согласие | Обязательно | Описание |
|----------|-------------|----------|
| Обработка ПД | Да | 152-ФЗ, хранение и использование данных |
| Условия платформы | Да | Пользовательское соглашение |
| Комиссия 4% | Да | Согласие на комиссию платформы |
| Раздел комиссии | При сплите | Согласие на распределение |

### Защита данных (152-ФЗ)

- ФИО, телефон, email — шифруются (AES-256)
- ИНН — шифруется + валидируется через ФНС
- Все действия логируются (audit trail)

## Механизм споров

### Когда можно оспорить

В течение hold period (до подтверждения оказания услуги):
- Услуга не оказана
- Качество услуги не соответствует
- Сумма не соответствует договору
- Дублирующий платёж
- Несанкционированный платёж

### Процесс разрешения (Dispute Resolution)

```
Клиент открывает спор
        ↓
Агентство рассматривает (24 часа)
        ↓
[Решено] или [Эскалация на платформу]
        ↓
Платформа выносит решение (Arbitration)
        ↓
Возврат (Refund) или Выплата
```

### Доказательства

Стороны могут загружать:
- Скриншоты переписки
- Фото объекта
- Акты выполненных работ
- Договоры и приложения

## Интеграции

### T-Bank Safe Deal

**Что делает банк:**
- Принимает платёж на номинальный счёт
- Hold period до подтверждения услуги
- Распределяет средства после confirmation
- Выплачивает на счета участников

**Преимущества:**
- Hold period защищает клиента
- Деньги выплачиваются только после подтверждения услуги
- Dispute resolution при спорах
- Прозрачное распределение

### SMS.RU

- Отправка ссылок на оплату
- Приглашения со-агентам
- Коды подтверждения (OTP)
- Уведомления о статусах

### ФНС

- Проверка статуса самозанятого (НПД)
- Валидация ИНН
- Проверка по чёрным спискам

## Статусы сделки

```
Черновик
    ↓
Ожидает подписей
    ↓
Подписан
    ↓
Счёт выставлен
    ↓
Ожидает оплаты ──→ Ошибка оплаты
    ↓
Hold period ──→ Спор (Dispute) ──→ Возврат (Refund)
    ↓
Подтверждение услуги (Service Confirmation)
    ↓
Готов к выплате
    ↓
Выплата в процессе
    ↓
Завершён
```
