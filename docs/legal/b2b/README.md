# Пакет B2B-документов для онбординга агентств и исполнителеи LK Housler

**Сформировано:** 19.01.2026  
**Оператор:** ООО «Сектор ИТ»  
**URL:** https://lk.housler.ru

## 0) Что внутри

- b2b_offer.md — Публичная оферта/договор ИТ-услуг (B2B)
- assignment_instruction.md — Поручение/согласие на удержание и перечисление вознаграждения Платформы
- kyc_eligibility_policy.md — Регламент KYC/допуска исполнителеи
- dpa_processing_order.md — DPA/поручение на обработку ПД (для агентств)
- data_retention_policy.md — Политика хранения документов и логов
- audit_trail_terms.md — Соглашение об ЭДО и хранении юридически значимых логов
- README.md — эта инструкция

## 1) Где должны лежать в продукте (рекомендуемые URL/разделы)

Публичные страницы (доступ без авторизации) — если вы хотите давать ссылки заранее:
- /legal/b2b/offer
- /legal/b2b/assignment
- /legal/b2b/kyc
- /legal/b2b/dpa
- /legal/b2b/retention
- /legal/b2b/audit-trail

Внутри онбординга агентства/исполнителя (обязательно):
- шаг «Подключение» → показать ссылки на Offer + Assignment + KYC (+ DPA для агентства)
- кнопка «Подключить» активна только после проставления обязательных чекбоксов.

## 2) Как подтверждаются пользователями (акцепт) и где это происходит

### 2.1 Исполнитель/агентство при подключении (обязательные чекбоксы)

1) «Принимаю Публичную оферту (ИТ-услуги)» (b2b_offer.md)  
2) «Даю поручение на удержание и перечисление вознаграждения Платформы» (assignment_instruction.md)  
3) «Согласен(на) с Регламентом допуска/KYC» (kyc_eligibility_policy.md)  
4) Для агентства дополнительно: «Поручаю обработку ПД клиентов (DPA)» (dpa_processing_order.md)

### 2.2 Любой подписант при первом подписании документа (обязательный чекбокс)

- «Принимаю правила ЭДО и хранения юридически значимых логов» (audit_trail_terms.md)

### 2.3 Политика хранения

Ее обычно не требуют как отдельный чекбокс. Достаточно:
- ссылка из B2B оферты;
- доступ в кабинете агентства;
- запись версии в момент принятия оферты (как приложений).

## 3) Как логировать акцепты (минимальный набор полеи)

Для каждого акцепта хранить:
- subject_type: user/agency
- subject_id
- doc_type: offer/assignment/kyc/dpa/audit_trail/retention_bundle
- doc_version
- doc_url
- accepted_at (UTC)
- ip
- user_agent
- channel (web/mobile)
- evidence: (например, sms_verified=true, method=checkbox)

Важно: лог должен быть неизменяемым (append-only) или иметь историю версий.

## 4) Что нужно заполнить перед публикацией

Во всех документах есть поля [УКАЗАТЬ]. До публикации обязательно заполнить:
- ИНН/ОГРН/адреса ООО «Сектор ИТ»
- emails (support, legal notices)
- ссылка на список субпроцессоров (или перечислить их в DPA/Privacy)
- тарифы (процент/фикс), или ссылка на тарифную страницу в продукте
- фактические сроки хранения (если вы их утверждаете и фиксируете)
